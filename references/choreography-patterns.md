# Choreography Patterns

## Choreography Basics

Choreography is the designed transfer of attention across time. For each animated beat, specify:

- Entry: how the element becomes relevant.
- Action: what changes.
- Transfer: where the viewer should look next.
- Settle: how the frame resolves.

If a movement does not create meaning or transfer attention, remove it or make it quieter.

## Attention Ladders

Use an attention ladder when multiple elements enter:

1. Establish the container or context.
2. Reveal the focal object.
3. Add secondary evidence.
4. Add detail labels or micro-interactions.
5. Resolve to the takeaway.

Do not reveal details before the viewer knows what system they belong to.

## Staggering

Staggering should express relationship, not randomness.

Good reasons to stagger:

- reading order
- causality
- priority
- geographic or spatial order
- workflow order
- grouped similarity

Typical premium stagger ranges:

- Micro UI details: 20-60 ms apart.
- Related cards or list items: 50-120 ms apart.
- Distinct idea groups: 150-300 ms apart.

Avoid perfectly even mechanical staggering for organic systems. Use tighter clusters followed by small pauses.

## Anticipation and Settle

Use anticipation before meaningful changes:

- slight lift before expansion
- tiny dim before highlight
- soft scale-down before snap-open
- cursor hover before click
- line tension before draw

Use settle after meaningful changes:

- short overshoot only when brand permits
- opacity completes after position
- shadow stabilizes after movement
- type locks to baseline
- glow or highlight fades last

Premium motion often has small anticipation and clean settle, not exaggerated bounce.

## Transition Grammar

Prefer transitions with memory:

- Expand: a small object becomes the next scene.
- Collapse: a complex scene compresses into a simple object.
- Match cut: shape, line, or UI region matches between shots.
- Wipe by object: an object passes through frame and reveals the next state.
- Camera reveal: a move exposes new context or scale.
- Morph by meaning: chart becomes timeline, cursor path becomes workflow, text underline becomes connector.

Avoid transitions that only signal "next slide."

## Camera Language

Use camera motion with intent:

- Push in: increase importance, reveal detail, create confidence.
- Pull back: show system, scale, or consequence.
- Lateral move: compare related states or move through workflow.
- Tilt or orbit: reveal depth, but use sparingly.
- Snap cut: create product confidence or speed.
- Hold: signal that the idea deserves attention.

Do not keep the camera moving to hide weak composition. A strong frame can be still.

## Layered Motion

Use three motion layers:

- Primary motion: the thing that explains the beat.
- Secondary motion: supports spatial context or product detail.
- Ambient motion: light, grain, cursor blink, subtle data activity.

Primary motion should be the most legible. Ambient motion should never compete with readable UI or headline type.

## Cause and Effect

Premium SaaS motion often feels intelligent because cause and effect are visible:

- command issued -> system responds
- input arrives -> model organizes
- decision selected -> workflow updates
- teammate comments -> status changes
- integration connects -> data appears
- permission set -> access boundary updates

Show the cause before the effect unless mystery is the point of the hook.

## Reveal Strategies

Use reveal types based on meaning:

- Mask reveal: controlled disclosure, precision, product polish.
- Draw-on reveal: process, path, setup, timeline, connection.
- Depth reveal: platform breadth, object importance, premium spatiality.
- Light reveal: intelligence, focus, activation.
- Assembly reveal: construction, modularity, integration.
- Replacement reveal: transformation, simplification, before-after.

## Continuity Rules

- Preserve direction unless reversing direction has meaning.
- Keep related objects in consistent spatial zones.
- Do not move the viewer's eye to a corner and cut to the opposite corner without a bridge.
- Do not cut away before the viewer reads the result.
- Use a final motion vector to point toward the next focal point.
