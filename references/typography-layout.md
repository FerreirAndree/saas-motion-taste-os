# Typography and Layout

## Typography Role

Type in premium SaaS motion should feel authored, not decorative. It can:

- frame the promise
- punctuate a reveal
- label product evidence
- create rhythm
- bridge between abstract and product shots
- land the final memory

It should not explain everything the visuals failed to communicate.

## Copy Density

Use short, confident copy:

- Hero headline: 2-7 words.
- Support line: 6-14 words.
- Label: 1-4 words.
- UI annotation: 2-6 words.

Avoid full sentences during fast visual motion. Let dense copy appear only during slower holds.

## Hierarchy

Use three type levels:

- Hero: single big thought, strong contrast, enough breathing room.
- Functional: product labels, section names, UI callouts.
- Micro: metadata, timestamps, values, hints.

Do not let all type levels compete. If UI text is dense, hero type must be quieter or absent.

## Layout Principles

- Use alignment as a premium signal.
- Keep generous margins around hero text.
- Balance product surfaces and type by optical weight, not equal area.
- Let type lock to a grid or product edge.
- Avoid centering everything by default.
- Use asymmetry when it creates direction and hierarchy.
- Use full-frame negative space for important claims.

## Type Motion

Choose type motion by meaning:

- Fade and rise: calm introduction.
- Masked slide: precision, editorial confidence.
- Character or word stagger: emphasis, but use sparingly.
- Scale lock-in: arrival, importance, brand resolve.
- Tracking expansion: spacious premium tone, but avoid overuse.
- Line reveal: connection to systems, charts, or workflows.

Keep type baselines stable. Avoid wobbly or elastic type unless the brand is playful.

## Readability Timing

Give viewers enough time:

- Short headline: 1.0-1.8 seconds.
- Support line: 1.8-3.0 seconds.
- Dense UI proof: 2.5-5.0 seconds depending on complexity.

If the camera is moving, increase hold time or reduce copy.

## Annotation Rules

Annotations should behave like intelligent product overlays:

- point exactly at the feature
- avoid covering important UI
- arrive after the product region is established
- leave before the next focal action
- use consistent pointer, line, or pill styles

Do not annotate every feature. Annotate what the viewer would otherwise miss.

## Premium Type Mistakes

- Giant generic headline with unrelated abstract background.
- Multiple large messages in one frame.
- Captions that duplicate voiceover word for word.
- Type moving while the viewer must read dense UI.
- Weak line breaks.
- Type too close to frame edges.
- Labels with no visual target.
- Random font mixing.
