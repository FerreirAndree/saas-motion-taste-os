# Timing and Rhythm

## Timing Principles

Premium motion feels intentional because timing separates ideas. The viewer should feel:

- anticipation before a meaningful change
- action during the change
- settle after the change
- breath before the next idea

Do not run every beat at the same speed.

## Duration Ranges

Use these as starting points:

- Micro-interaction: 120-250 ms.
- UI state change: 200-450 ms.
- Card entrance or exit: 300-700 ms.
- Camera push or pull: 900-1800 ms.
- Product feature proof: 2-5 seconds.
- Hero reveal: 2-4 seconds.
- Final brand hold: 1.5-3 seconds.

Shorter is not always more premium. Important ideas need time.

## Easing Taste

Prefer easing that feels precise:

- Fast-out, soft-in for UI entrances.
- Ease-in-out for camera moves.
- Slight overshoot only for friendly or tactile brands.
- Linear only for mechanical movement, data scanning, or constant systems.
- Hard cuts for confidence and speed when continuity is already clear.

Avoid bouncy defaults, mushy ease, and movements that take too long to settle.

## Rhythm Shapes

### Calm Confidence

- medium-length shots
- clean holds
- quiet transitions
- low motion density

Use for enterprise, security, finance, infrastructure, and premium productivity.

### Product Momentum

- quick proof beats
- clear UI actions
- frequent but motivated transitions
- occasional hard cuts

Use for launch videos with many capabilities.

### Intelligent Flow

- continuous attention handoff
- data or UI elements cause each next action
- smooth but not sleepy

Use for AI, analytics, workflow, and automation.

### Editorial Punch

- strong typographic statements
- abrupt cuts
- contrast between stillness and fast movement

Use for category-defining claims or bold brands.

## Pauses

Use pauses as taste tools:

- after the product first appears
- after a hard problem is stated
- after a surprising result
- before the final brand resolve
- when a dense UI proof needs reading time

A pause can make the product feel more confident than another animation would.

## Motion Density

Control the number of moving things:

- Hook: high contrast or one striking motion.
- Explanation: medium density, clear hierarchy.
- Proof: low density around the focal UI action.
- Montage: high density only if the sequence is rhythmic and brief.
- Trust: low density, slow controlled motion.
- Resolve: minimal motion.

## Music and Sound Awareness

Even when not producing audio, design as if sound exists:

- land visual reveals on implied beats
- use micro-pauses where sound design would punctuate
- let transitions have attack and decay
- avoid visual rhythms that would fight a voiceover

## Pacing Diagnostics

If a sequence feels cheap or rushed:

- reduce simultaneous motion
- lengthen the hold after key reveals
- cut transitional filler
- make the first half less explanatory and more visual
- give product proof more screen time

If a sequence feels slow:

- remove redundant setup
- combine adjacent weak beats
- add clearer cause-effect actions
- use sharper cuts between already-understood states
- increase contrast between quiet and active moments
